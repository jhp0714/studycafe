내 정보기능에서 비밀 번호 확인 삭제


1. 좌석 및 자원 유형
    1. 좌석 및 자원은 다음 세 가지로 구분된다.
        1. 일반석
        2. 지정석
        3. 사물함
2. 상품
    1. 판매 상품은 시간제와 기간제로 구분된다.
        1. 지정석, 사물함 : 기간제로만 판매
        2. 일반석 : 기간제, 시간제 모두 판매 가능
    2. 주문은 항상 한 상품만 가능하다.(수량 == 1)
3. 좌석 배정 및 이동 공통 규칙
    1. 일실, 지정석, 사물함 배정, 좌석 이동은 비어있는 좌석 또는 사물함이 있는 경우에만 가능하다.
    2. 좌석 선택 및 이동 시
        1. 이용 가능 시간, 퇴실 예정 시간은 변경되지 않는다.
4. 입실 및 이용 가능 시간
    1. 지정석을 제외한 모든 좌석은 입실 시 이용 가능 시간이 제한된다.
        1. 남은 시간이 12시간 미만인 경우 : 남은 시간만큼 이용 가능
        2. 남은 시간이 12시간 이상인 경우 : 최대 12시간 이용 가능
    2. 기간제 일반석 또한 동일하게 12시간 이용 가능 시간 제한이 적용된다.
5. 퇴실
    1. 일반석(normal seat)인 경우
        1. 이용 가능 시간이 종료되면 자동 퇴실 처리된다.
        2. 수동 퇴실을 선택한 경우
            1. 시간제 상품(time) : 남은 시간이 저장된다.
            2. 기간제 상품(flat) : 시간은 저장되지 않으며, 결제 기준으로 계속 흐른다.
        3. 체크 아웃 시 해당 SeatUsage는 행 삭제한다.
    2. 지정석(fixed, seat), 사물함(locker)인 경우
        1. 퇴실 개념이 없다.
        2. 결제 완료 시점부터 점유 시작이며, 만료/환불/이동 시에만 점유가 종료되거나 바뀐다.
6. 시간 연장
    1. 시간 연장은 일반 좌석에 대해서만 가능하다.
    2. 시간 연장 시
        1. 퇴실 예정 시간 계산 후에도 상품 시간이 남아 있다면 한 번에 최대 6시간 까지 퇴실 예정 시간이 추가된다.
        2. 상품 시간이 남아 있지 않다면 추가 결제를
        3. 시간 연장은 한 번에 최대 6시간 가능하며, 남은 시간이 있는 경우 여러 번 반복하여 연장 가능하다.
        4. 상품의 만료 시간과 퇴실 예정 시간은 서로 다른 개념이다.
            1. 퇴실 예정 시간은 입실 시 생성되며, 시간 연장 시 갱신된다.
    3. 지정석의 경우 시간 연장은 개념은 없고 추가 결제를 통한 기간 연장만 가능하다.
7. 지정석/사물함 정책
    1. 지정석/사물함은 결제 기간 동안 고정 배정 상태이며, 결제 완료 시점부터 항상 점유 상태로 본다.
    2. 지정석/사물함 선택은 점유 이동이 아닌 경우 첫 결제 시에만 진행된다.
        1. 첫 결제 시 비어있는 지정석/사물함을 선택해야 결제가 완료된다.
        2. 이미 지정석/사물함을 보유한 사용자는 추가 결제 시 좌석을 다시 선택하지 않으며, 기존 pass의 종료일만 연장된다.
    3. 빈 지정석이 없는 경우 지정석/사물함 상품은 구매불가가된다.
        1. 지정석 좌석이 모두 점유 중이면 지정석/사물함 상품은 구매/결제 불가 상태로 노출한다.
        2. 빈 지정석/사물함이 발생하면 지정석/사물함 상품은 자동으로 활성화된다.
    4. 지정석/사물함 이용 중 빈 지정석/사물함이 있으면 이동 가능하고 이 경우 기존 지정석 점유는 종료되고 새 지정석 점유가 시작되며
     하루 12시간 + 6시간 연장 규칙은 적용되지 않는다.
    5. 지정석/사물함은 입실/퇴실 개념 없이 배정/유지/이동/종료 개념으로 관리한다.
8. 시간 흐름 기준
    1. 시간제 상품
        1. 입실 후 ~ 퇴실 전까지 시간 감소
        2. 퇴실 시 남은 시간 저장
    2. 기간제 상품 / 지정석 / 사물함
        1. 결제 시점부터 시간이 지속적으로 흐른다.
        2. 지정석/사물함은 결제 시점부터 점유가 시작되며, 별도 체크인 없이 유지된다.
9. 좌석 풀 분리
    1. 지정석과 일반석은 물리적으로 분리되며 서로의 좌석을 공유하지 않는다.
10. 다중 상품 보유 시 우선 순위
    1. 기간제와 시간제를 동시에 보유한 경우
        1. 기간제 상품이 우선 사용된다.
        2. 시간제 상품은 일반석 시간제 pass로 입실 중일 때만 소모된다.
    2. 지정석 사용 중에도
        1. 일반석 기간제 상품의 시간은 정상적으로 소모된다.
        2. 일반석 시간제 상품은 일반석 시간제 pass로 입실 중일 때만 소모된다.
11. 사용자 점유 제한
    1. 한 사용자는 동시에 좌석 1개, 사물함 1개만 사용할 수 있다.
    2. 지정석과 일반석을 동시에 점유할 수 없다.
12. 자동 퇴실 후 재입실
    1. 자동 퇴실 후 남은 시간이 있다면 재입실 가능하다.
    2. 재입실 시 다시 12시간 이용 가능 규칙이 적용된다.
13. 인증 정책
    1. 탬색성 조회는 비회원도 가능하다.
        1. 좌석 목록, 사물함 목록, 상품 목록 조회
    2. 실제 서비스 이용은 회원만 가능하다.
    3. 관리자는 별도 가입 절차 없이 db에서 설정한다.
        1. 관리자는 DB/shell에서 is_admin=True만 주는 방식
    4. 로그인 정책
        1. 아이디 : 전화번호
        2. 비밀번호 : 전화번호 뒷자리 4자리
14. 결제 정책
    1. pg 연동 없이 다음 상태를 분리하여 관리한다.
        1. order, payment, pass
15. 내 정보 기능
    1. 사용자는 내 정보 화면에서 다음 항목을 확인할 수 있다.
        1. 아이디
        2. 결제된 상품
        3. 현재 사용 중인 좌석
16. 관리자 권환
    1. 좌석, 지정석, 사물함 생성/비활성화
    2. 사용자 강제 퇴실
        1. 강제 퇴실으 수동 퇴실과 동일한 시간 처리 정책을 따르며, 로그에 강제 퇴실로 기록된다.
    3. 결제 내역, 이용 내역 조회
    4. 환불 기능
17. 로그
